Week 1: Sorting Books

Week of 2/16/2020 - 2/23/2020

Available languages: Java 11.0.5, Python 3.7.2, GNU G++17 7.3.0, C# Mono 5.18, Kotlin 1.3.10, PHP 7.2.13

You are in charge of ordering books 8 books on a shelf in a library: Candyland, Central Data, Cinderhulk, Clownman Begins, CMOS Integration, Codename: Monkey, Cute Dogs, and Cynthia's Story. However, the head librarian has demanded that certain books be next to each other. Because of state library standards, books must be as close to alphabetical order as possible. Given this information, please determine the lexicographically smallest permutation of books possible, given an n number of instructions (demands).

INPUT:
The first line of the input is n. The next n lines describe a demand from the head librarian. The format of the demand is as such: "X must be placed beside Y", where X and Y are the titles of books.

OUTPUT:
The output consists of 8 lines, each with the title of a book. The order should satisfy all constraints. If there are muliple solutions, output the one that alphabetically comes first.

TEST INPUT:

3
Cute Dogs must be beside Cinderhulk.
Codename: Monkey must be beside Cinderhulk.
Cynthia's Story must be beside Candyland.

TEST OUTPUT:

Candyland
Cynthia's Story
Central Data
Clownman Begins
CMOS Integration
Codename: Monkey
Cinderhulk
Cute Dogs

Problem Credits: USACO 2019 December Contest, Bronze; adapted by Sid Banerjee, original problem by Brian Dean here.

Note: In the "official" solution to this problem, Nick Wu uses the built-in C++ std::next_permutation to automatically traverse through the lexicographically ordered permutations of cows (or in this case, books). He can implement complete search because 8! will run well under the time limit (8! is 40320). For this reason, I recommend finding a solution in another language, as C++ has an "easy" solution.
