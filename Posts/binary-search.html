Posted on April 3, 2020 at 6:42 PM [PST]

What is Binary Search?

Binary search is an algorithm for finding data in a set. Binary search runs in, at worst, logarithmic time. Thus, the complexity of the search is O(log n). Today we look at the recursive approach to this algorithm.

Imagine an array of 9 sorted integers. Your array may look like the following:

int[] arr = {1, 3, 6, 8, 10, 14, 18, 21, 24};

Binary search finds the central element (in this case, it would be 10), and checks to see whether it is greater than, less than, or equal to the target value. Let's say our target value is 18. Since 10 is less than 18, we know that our target value must be in the second half of the array. In other words, our target (18) is guaranteed to be to the "right" of the number we checked (10). This narrows down our array to the following numbers:

// {14, 18, 21, 24};

Since we have an even number of elements (and our method for locating the central element uses integers), the new "central" element will be 18. The formula for locating the central element is as follows:

int mid = startPos + (endPos - startPos) / 2;

Each time the function is called, the three checks occur. If the target value is less than the central value, then narrow down the search to the left half. If the target value is equal to the central value, return the position it is in. Finally, if the target value is greater than the central value, narrow down the search to the right half of the remaining array. The code for this is such:

static int binarySearch(int[] arr, int s, int e, int target){

    int mid = s + (e - s) / 2;

    if(e >= s){
        if(mid < target){
            return binarySearch(arr, mid+1, arr.length, target);
        }

        if(mid == target){
            return mid;
        }

        if(mid > target){
            return binarySearch(arr, 0, mid+1, target);
        }
    }

    return -1; // if the element does not exist, return -1.
}

Algorithms with logarithmic complexity are some of the fastest methods. Your runtime will be considerably faster if you implement binary search because logarithmic algorithms increases at a lower rate than linear-complexity algorithms.
